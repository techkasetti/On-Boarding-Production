# NA-02 Candidate Intake and Profile Persistence Runbook

## Purpose
Create and maintain candidate profile completeness.

## Preconditions
1. Intake flow active
2. Profile fields available in page layout
3. Candidate permissions assigned

## Step-by-Step Operation
1. Launch intake process.
2. Populate personal/professional details.
3. Save profile and linked records.
4. Review profile hub summary.

## Validation Checklist
1. Required fields complete
2. Related records linked correctly
3. No validation errors

## Failure Handling and Recovery
1. Correct invalid data and resubmit
2. Review CandidateIntakeController logs
3. Escalate mapping defects

## Ownership
1. Product/Operations Owner: business correctness and sign-off.
2. Salesforce Admin: configuration, visibility, and flow health.
3. Apex/Integration Owner: runtime stability, logs, retries.
